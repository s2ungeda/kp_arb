# 괴리 보드 ↔ 엑셀(IM.xlsx) 좌표 매핑 — 검증용

모니터의 "괴리 보드" 값과 엑셀 값을 나란히 대조하기 위한 표.
(값은 수신 시점 차이로 순간적으로 다를 수 있음 — 방향과 크기가 일치하면 정상)

## 1. 괴리 보드 표의 각 칸

| 모니터 칸 | 뜻 | 엑셀 (하이닉스 기준) |
|---|---|---|
| HL매도d | HL 매도호가 환산가의 주식 대비 괴리 | 시세!**AE7** (= 메인!J32 "ask disp") |
| HL매수d | HL 매수호가 환산가의 주식 대비 괴리 | 시세!**AF7** (= 메인!J33 "bid disp") |
| 국내매도d (SF행) | 선물 매도호가의 선물이론가 대비 괴리 | 시세!**AE61** (= 메인!K32) |
| 국내매수d (SF행) | 선물 매수호가의 선물이론가 대비 괴리 | 시세!**AF61** (= 메인!K33) |
| 국내매도d (ETF행) | ETF 매도호가의 ETF이론가 대비 괴리 | 시세!**AE89** (KODEX 0193T0 = 메인!M32) |
| 국내매수d (ETF행) | ETF 매수호가의 ETF이론가 대비 괴리 | 시세!**AF89** (= 메인!M33) |
| **진입** (SF행) | HL매수d − 국내매도d | 메인!**K22** ("HL-SF 상대호가진입") |
| **청산** (SF행) | HL매도d − 국내매수d | 메인!**K24** |
| **진입** (ETF행) | 〃 (ETF 상대) | 메인!**M22** (KODEX) / **N22** (TIGER) |
| **청산** (ETF행) | 〃 | 메인!**M24** / **N24** |

단위: 모니터는 % 표시(×100), 엑셀 메인 시트는 소수(예: -0.00695 = -0.695%).

## 2. 종목별 엑셀 행 번호 (시세 시트)

| 종목 | HL (AE/AF행) | 주식 | 주식선물 | ETF KODEX | ETF TIGER |
|---|---|---|---|---|---|
| SK하이닉스 | 7행 | 60행 | 61행 | 89행 (0193T0) | 91행 (0195S0) |
| 삼성전자 | 8행 | 58행 | 59행 | 88행 (0193W0) | 90행 (0195R0) |
| 현대차 | (없음) | 62행 | 63행 | — | — |

※ 시스템은 ETF를 **config.yaml에 적은 1종**만 계산 (현재 KODEX). TIGER로 바꾸려면 yaml 수정.

## 3. 중간 계산값 대조 (값이 안 맞을 때 원인 추적용)

| 값 | 시스템 | 엑셀 |
|---|---|---|
| 환율이론가 | `LiveSystem.usdkrw_theory` (원달러선물×1.5% 캐리) | 시세!**I1** (=E65) |
| 원달러선물 현재가 | t2111 조회 | 시세!**D65** |
| 기초 주식 현재가(KRX) | trades[(u, 주식, krx)] | 시세!**AA60**(하이닉스)/AA58(삼성) |
| 선물이론가 (3.5% 캐리) | `stock_futures_theory()` | 시세!**AG61**(=E61)/AG59 |
| ETF 이론가(iNAV) | `etf_theory_price()` | 시세!**AG89**(=D89)/AG88 |
| HL 매도/매수 1호가 | HL bbo | 시세!**F7/D7** |

## 4. 예상되는 사소한 차이

- **국내 호가**: 시스템은 KRX+NXT 통합 최우선호가(운영 키일 때). 모의는 KRX만.
- **HL 1호가**: 엑셀 RTD는 호가단위(K7=0.5)로 묶은 호가, 시스템은 원시 bbo — 한 틱 차이 가능.
- **선물이론가**: 양쪽 다 "기초 현재가 × (1+3.5%×잔존일/365)" 동일 공식이지만 기초 체결가 수신 시점 차이만큼 어긋날 수 있음.
- 장 시작 직후: 시스템은 기초 체결(S3_)이 최소 1건 와야 이론가 계산 시작 (엑셀도 동일).
