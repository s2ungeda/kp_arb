# 괴리 보드 ↔ 엑셀(meme.xlsx) 좌표 매핑 — 검증용

모니터의 값과 엑셀(개정판 meme.xlsx) 값을 나란히 대조하기 위한 표.
(값은 수신 시점 차이로 순간적으로 다를 수 있음 — 방향과 크기가 일치하면 정상)
ETF는 취급 제외(2026-07-13) — 쌍은 **HL vs 주식선물(SF)** + **HL vs 주식(S)**.
주식 쌍(엑셀에 공식 없음 — 시스템 유도): 기준가 = 주식 자기 현재가(옛 엑셀 현대차
행 AE62/AF62 패턴), 진입/청산 공식은 SF와 동일. 방향 A(주식 매수+HL 숏) 전용,
주식 잔고 보유 시 운용. 주식 행의 주선차는 정의상 항상 0이라 '-' 표시.

## 1. 괴리 보드 표의 각 칸

| 모니터 칸 | 뜻 | 엑셀 (하이닉스 기준) |
|---|---|---|
| (LH차 — 화면 제외) | 국내 다리 현재가 대비 HL 현재가 괴리차. CSV의 hl_last_d − kr_last_d로 재계산 가능 | 시세!**AD7** − 시세!**AD61** (= 메인!J12 − L9. S행은 AD7만) |
| **진입** | **HL매수d − 국내매수d** (국내 maker: bid에 걸어 매수 + HL bid 매도) | 메인!**L12** (en, =J23−K23) |
| **청산** | **HL매도d − 국내매도d** (국내 ask에 걸어 매도 + HL ask 환매수) | 메인!**L14** (ex, =J22−K22) |
| **순진입** | 진입 − (청산−진입)/2 − 왕복수수료 — 완전 수렴 시 기대 %. **≥0 진입** | (엑셀 대응 없음 — 신규) |
| **순청산** | (진입+청산)/2 — 중간가 기준 괴리. **≤0이면 수렴 완료 → 청산** | (엑셀 대응 없음 — 신규) |
| HL매도d/HL매수d | HL 매도/매수호가 환산 괴리 | 시세!**AE7/AF7** (= 메인!J22/J23) |
| 국내매도d/국내매수d | 선물 매도/매수호가의 이론가 대비 괴리 | 시세!**AE61/AF61** (= 메인!K22/K23) |

단위: 모니터는 % 표시(×100), 엑셀은 소수(예: 0.00075 = +0.075%).
구성요소 열(HL/국내 매도·매수d)은 화면에서 제외 — CSV(logs/spread_*.csv)에는 계속 기록.

## 2. LS 표의 이론가·괴리율·예상체결가

| 모니터 칸 | 뜻 | 엑셀 |
|---|---|---|
| 이론가 (선물행) | 기초 현재가 × (1 + 3.5% × 잔존일/365) | 시세!**E61**/E59 (RTD 선물이론가) |
| 괴리율% | (현재가 − 이론가) ÷ 이론가 × 100 | 시세!AD61×100 |
| 예상체결가 (주식/선물) | 동시호가 예상체결 (YS3·UYS / YJC) | 시세!**K60/K61** |

## 4. 중간 계산값 대조 (값이 안 맞을 때 원인 추적용)

CSV 기록(logs/spread_날짜.csv) 열:
`hl_ask_d, hl_bid_d, kr_ask_d, kr_bid_d, entry, exit, usdkrw_theory, base_last, hl_last_d, kr_last_d, net_entry, net_exit`

| 값 | 시스템 | 엑셀 |
|---|---|---|
| 환율이론가 | `usdkrw_theory` (CSV usdkrw_theory) | 시세!**I1** |
| 원달러선물 현재가 | `usdkrw_futures` (상태줄 환율) | 시세!**D65** |
| 기초 주식 현재가 | `stock_last()` — 통합(uni) 우선→KRX (CSV base_last) | 시세!**AA60** (=D60, RTD 현재가) |
| 선물이론가 (3.5% 캐리) | `stock_futures_theory()` | 시세!**AG61**(=E61) |
| HL 매도/매수 1호가 | HL bbo | 시세!**F7/D7** |
| HL 현재가 괴리 | CSV hl_last_d | 시세!**AD7** (=메인 J12) |
| 국내 현재가 괴리 | CSV kr_last_d | 시세!**AD61** (=메인 L9) |

## 5. 예상되는 사소한 차이

- **HL 1호가**: 엑셀 RTD는 호가단위(0.5/0.05)로 묶은 호가, 시스템은 원시 bbo — 한 틱(~0.03%) 차이 가능. 진입/청산이 평행하게 어긋나는 주요 원인 후보.
- **수신 시점**: 양쪽 화면 갱신 주기(모니터 0.3초, 엑셀 RTD 스로틀)만큼 순간 차이.
- 동시호가 변형(엑셀 L16/L17 — 예상체결가 기반)은 미구현(추후).
